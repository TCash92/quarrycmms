# ROI Calculation Template

> **Purpose**: Calculate return on investment for QuarryCMMS by comparing baseline metrics against post-launch performance. Complete this at Day 60 of the pilot.

---

## When to Complete This Template

- **Day 60** of pilot (after 8 weeks of app usage)
- Requires: 2-week baseline data + 8-week app data
- Output: Dollar savings and ROI percentage

## Calculation Examples

### Example 1: Downtime Savings

```
Baseline: 15 hours/week downtime
Pilot: 10 hours/week downtime
Reduction: 5 hours/week

Site value: $500/hour

Weekly savings: 5 x $500 = $2,500
Annual savings: $2,500 x 52 = $130,000
```

### Example 2: Labor Efficiency

```
Baseline avg WO time: 4.5 hours
Pilot avg WO time: 3.0 hours
Time saved: 1.5 hours/WO

WOs per week: 25
Hours saved: 1.5 x 25 = 37.5 hours/week

Tech rate: $45/hour

Weekly savings: 37.5 x $45 = $1,687
Annual savings: $1,687 x 52 = $87,750
```

### Example 3: Full ROI

```
Annual Benefits:
  Downtime:    $130,000
  Labor:       $87,750
  Duplicates:  $15,000
  Paperwork:   $25,000
  Compliance:  $10,000
  ─────────────────────
  TOTAL:       $267,750

Annual Costs:
  Software:    $12,000
  Hardware:    $5,000
  Training:    $3,000
  ─────────────────────
  TOTAL:       $20,000

Net Savings: $247,750
ROI: 1,238%
Payback: 0.9 months
```

## Conservative Estimates

When estimating values, use **conservative** numbers:

| Value | Conservative Approach |
|-------|----------------------|
| Downtime cost | Use minimum hourly value |
| Labor rate | Use base pay, not overtime |
| Compliance savings | Use lowest historical cost |
| Improvement % | Use lower bound of range |

Conservative estimates make ROI credible. If actual results beat estimates, that's a bonus.

## Presenting Results

### To Site Management

Focus on:
- Net dollar savings
- Payback period
- Non-quantified benefits (visibility, morale)

### To Corporate

Focus on:
- ROI percentage
- Scalability to other sites
- Compliance risk reduction

### To Technicians

Focus on:
- Time saved on paperwork
- Fewer duplicates and confusion
- Better visibility of workload

---

## Troubleshooting Low ROI

If ROI calculation shows low return:

| Issue | Possible Cause | Solution |
|-------|---------------|----------|
| Small downtime reduction | Already efficient OR not capturing all downtime | Check baseline accuracy |
| No WO time improvement | Complex underlying issues | May need process changes |
| High costs | Over-provisioned hardware | Review actual needs |

An honest low ROI is still valuable - it identifies where to focus improvements.
